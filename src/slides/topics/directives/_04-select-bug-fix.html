A subtle bug: <select> values can desync during initial render

  • In a template like <select model=\${...}>\${options}</select>, the binding part is visited before the child options part
  • That means the binding can set select.value before the matching <option> exists in the DOM
  • The fix is pragmatic: for HTMLSelectElement, sync immediately and then resync in a microtask after the render pass completes
  • This is exactly the kind of edge case a tiny runtime exposes very clearly
